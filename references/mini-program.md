## Recognition

A normal mini program normally contains or reconstructs:

- `app.json`, `app.js`, `app.wxss`
- `pages/...`
- page `.js`, `.json`, `.wxml`, `.wxss`
- `components/...`
- optionally `workers/`, `sitemap.json`, `ext.json`, plugins, subpackages, or cloud functions

Raw packages may instead expose `app-service.js`, `app-config.json`, `page-frame.html`, `page-frame.js`, `app-wxss.js`, and `workers.js` until an unpacker splits them.

## Reconstruction Order

1. Rebuild `app.json` and verify every page route.
2. Restore the corresponding page script, configuration, template, and style files.
3. Resolve `usingComponents` from page and global configuration.
4. Restore subpackages at their declared roots.
5. Resolve plugins, workers, custom tab bars, and independent packages.
6. Import the root containing `app.json`, not a nested extraction directory.
7. Fix the first compile error before changing business behavior.

## Page And Component Recovery

For each page route `pages/a/index`, verify:

```text
pages/a/index.js
pages/a/index.json
pages/a/index.wxml
pages/a/index.wxss
```

Some files may legitimately be absent because configuration or styles are inherited, but the page script and template must match the runtime format.

Check component references in:

- `app.json` global `usingComponents`
- each page/component JSON
- plugin component paths
- npm-built paths under `miniprogram_npm`

Do not rewrite every missing component as an empty component. Determine whether it is local, plugin-provided, generated by a framework, or absent from the captured package.

## Compiled Frameworks

An unpacked project is compiled output, not guaranteed source recovery:

- uni-app output does not reconstruct original Vue SFC files.
- Taro output does not reconstruct original React/TSX sources.
- WePY and other frameworks may retain runtime wrappers but lose source organization.
- source maps, if present, materially improve recovery; preserve them.

Aim first for a runnable WeChat project. Reconstructing the original framework project is a separate task and may be impossible without source maps.

## Platform Services

Trace these independently:

- `wx.login`, session exchange, user profile
- `wx.request`, `wx.uploadFile`, `wx.downloadFile`
- `wx.cloud.*`
- subscriptions and template messages
- maps, location, Bluetooth, camera, NFC, filesystem
- payment and authorization
- ads and analytics

For offline reconstruction, replace only the boundaries required by the requested flow. Do not fake permissions, payment success, or protected account access. Disable those flows with a clear local message.

Cloud functions and backend databases are usually not recoverable from the client package. Recreate only the response contracts needed for authorized local testing.

## Validation

- Open every requested page route.
- Verify navigation, back behavior, tab bar state, and component rendering.
- Trigger representative form/input actions.
- Test local storage across reload.
- Confirm network failures are expected or replaced; do not hide them.
- Preview on a real device for APIs that differ from DevTools.
